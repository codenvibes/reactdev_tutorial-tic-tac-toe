## Completing the game
<p class="whitespace-pre-wrap my-4">By this point, you have all the basic building blocks for your tic-tac-toe game. To have a complete game, you now need to alternate placing “X”s and “O”s on the board, and you need a way to determine a winner.

<h3 class="mdx-heading text-2xl font-display leading-9 text-primary dark:text-primary-dark font-bold my-6" id="lifting-state-up">Lifting state up <a aria-label="Link for Lifting state up " class="mdx-header-anchor inline-block" href="https://react.dev/learn/tutorial-tic-tac-toe#lifting-state-up" title="Link for Lifting state up "><svg class="text-gray-70 ms-2 h-5 w-5" height="1em" viewbox="0 0 13 13" width="1em" xmlns="http://www.w3.org/2000/svg"><g fill="currentColor" fill-rule="evenodd"><path d="M7.778 7.975a2.5 2.5 0 0 0 .347-3.837L6.017 2.03a2.498 2.498 0 0 0-3.542-.007 2.5 2.5 0 0 0 .006 3.543l1.153 1.15c.07-.29.154-.563.25-.773.036-.077.084-.16.14-.25L3.18 4.85a1.496 1.496 0 0 1 .002-2.12 1.496 1.496 0 0 1 2.12 0l2.124 2.123a1.496 1.496 0 0 1-.333 2.37c.16.246.42.504.685.752z"></path><path d="M5.657 4.557a2.5 2.5 0 0 0-.347 3.837l2.108 2.108a2.498 2.498 0 0 0 3.542.007 2.5 2.5 0 0 0-.006-3.543L9.802 5.815c-.07.29-.154.565-.25.774-.036.076-.084.16-.14.25l.842.84c.585.587.59 1.532 0 2.122-.587.585-1.532.59-2.12 0L6.008 7.68a1.496 1.496 0 0 1 .332-2.372c-.16-.245-.42-.503-.685-.75z"></path></g></svg></a></h3>

<p class="whitespace-pre-wrap my-4">Currently, each <code class="inline text-code text-secondary dark:text-secondary-dark px-1 rounded-md no-underline bg-gray-30 bg-opacity-10 py-px" dir="ltr">Square</code> component maintains a part of the game’s state. To check for a winner in a tic-tac-toe game, the <code class="inline text-code text-secondary dark:text-secondary-dark px-1 rounded-md no-underline bg-gray-30 bg-opacity-10 py-px" dir="ltr">Board</code> would need to somehow know the state of each of the 9 <code class="inline text-code text-secondary dark:text-secondary-dark px-1 rounded-md no-underline bg-gray-30 bg-opacity-10 py-px" dir="ltr">Square</code> components.

<p class="whitespace-pre-wrap my-4">How would you approach that? At first, you might guess that the <code class="inline text-code text-secondary dark:text-secondary-dark px-1 rounded-md no-underline bg-gray-30 bg-opacity-10 py-px" dir="ltr">Board</code> needs to “ask” each <code class="inline text-code text-secondary dark:text-secondary-dark px-1 rounded-md no-underline bg-gray-30 bg-opacity-10 py-px" dir="ltr">Square</code> for that <code class="inline text-code text-secondary dark:text-secondary-dark px-1 rounded-md no-underline bg-gray-30 bg-opacity-10 py-px" dir="ltr">Square</code>’s state. Although this approach is technically possible in React, we discourage it because the code becomes difficult to understand, susceptible to bugs, and hard to refactor. Instead, the best approach is to store the game’s state in the parent <code class="inline text-code text-secondary dark:text-secondary-dark px-1 rounded-md no-underline bg-gray-30 bg-opacity-10 py-px" dir="ltr">Board</code> component instead of in each <code class="inline text-code text-secondary dark:text-secondary-dark px-1 rounded-md no-underline bg-gray-30 bg-opacity-10 py-px" dir="ltr">Square</code>. The <code class="inline text-code text-secondary dark:text-secondary-dark px-1 rounded-md no-underline bg-gray-30 bg-opacity-10 py-px" dir="ltr">Board</code> component can tell each <code class="inline text-code text-secondary dark:text-secondary-dark px-1 rounded-md no-underline bg-gray-30 bg-opacity-10 py-px" dir="ltr">Square</code> what to display by passing a prop, like you did when you passed a number to each Square.

<p class="whitespace-pre-wrap my-4"><strong class="font-bold">To collect data from multiple children, or to have two child components communicate with each other, declare the shared state in their parent component instead. The parent component can pass that state back down to the children via props. This keeps the child components in sync with each other and with their parent.</strong>

<p class="whitespace-pre-wrap my-4">Lifting state into a parent component is common when React components are refactored.
